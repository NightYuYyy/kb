# kb — 个人知识库 Skill

## 触发条件（满足任一即触发）

- 用户说 **"记录到知识库"**、**"记录一下"**、**"存到 kb"**、**"录入"**
- 用户说 **"查知识库"**、**"搜一下知识库"**、**"kb 查"**
- 用户询问某个命令、脚本、工具的用法
- 用户提到 "知识库" 或 "kb"
- 对话中需要回顾之前记录的个人工具/命令

## 命令

`kb` 是全局命令，直接使用：

### 查知识库（语义检索 + AI 回答）

```bash
kb ask "用户的问题"
```

### 录入知识（AI 自动格式化为 Markdown + 提取标题标签）

```bash
kb add "要记录的内容"
```

### 查看条目详情

```bash
kb show <条目ID>
```

### 列出所有条目

```bash
kb list
kb list --tag 标签名
```

### 删除条目

```bash
kb delete <条目ID>
```

## 行为准则

1. **查知识库优先于猜测**：遇到 CLI 命令/工具用法问题，先 `kb ask`，不要凭记忆或训练数据瞎编
2. **听到"记录到知识库"立即执行**：用户说这句话时，把上下文中的命令/工具说明用 `kb add` 录入，不要犹豫
3. **知识库无结果时如实告知**：如果 `kb ask` 返回无相关内容，直接告诉用户知识库没有，可以建议录入
4. **回答时引用来源**：如果 `kb ask` 返回了多个来源，提及是从哪些条目获取的信息
5. **录入内容尽量完整**：把命令的所有用法、参数、路径依赖都写进去
