# kb — 个人知识库 Skill

当用户问 "怎么用 xxx 命令"、"xxx 工具怎么用"、"之前那个脚本的参数是什么"、
"记录一下这个命令"、"帮我查知识库" 等与 CLI 命令/工具/脚本用法相关的问题时，
**优先调用本 skill**。

## 触发条件

- 用户询问某个命令、脚本、工具的用法
- 用户说 "帮我查一下 xxx"
- 用户说 "记录 / 保存 / 录入 这个命令"
- 用户提到 "知识库" 或 "kb"
- 对话中明显需要回顾之前记录的个人工具/命令

## 命令

`kb` 已通过 `pip install -e .` 安装为全局命令，直接使用即可：

### 查知识库

```bash
kb ask "用户的问题"
```

### 录入知识

```bash
kb add "要记录的内容"
```

### 语义搜索

```bash
kb search "关键词"
```

### 列表浏览

```bash
kb list
kb list --tag 标签名
```

### 查看条目详情

```bash
kb show <条目ID>
```

### 删除条目

```bash
kb delete <条目ID>
```

## 行为准则

1. **查知识库优先于猜测**：遇到 CLI 命令/工具用法问题，先查 kb，不要凭记忆或训练数据瞎编
2. **知识库无结果时如实告知**：如果 `kb ask` 返回无相关内容，直接告诉用户知识库没有，可以建议录入
3. **录入时保持原文**：用户要求录入的内容原样保存，让系统自动提取元数据
4. **回答时引用来源**：如果 `kb ask` 返回了多个来源，提及是从哪些条目获取的信息
